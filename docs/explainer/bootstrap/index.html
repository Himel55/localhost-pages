bootstrap.py
seeds once.

An empty symlinks/ directory would mean an empty index — and a first-time user with nothing to look at. bootstrap.py solves that in 33 lines: on the very first run it symlinks the bundled docs (this site) into the registry, writes a marker, and never touches the directory again. It is the reason you are reading this through the server on a fresh checkout.

Why the default app is the docs

A brand-new checkout has an empty symlinks/, so the index would greet you with nothing. Seeding this explainer means the very first thing the server can show is a guided tour of itself — the docs are delivered by the machine they describe, with zero setup.

02 — the idempotency guard

Run once, ever — the marker decides.

The function's first act is to check for the marker and bail if it is there. Everything below the guard runs exactly once in the lifetime of a symlinks directory.

bootstrap.pylines 15 — 23
15def seed_defaults(symlinks_dir: Path) -> None:
16    """Register bundled default apps once, on first run.
17
18    Writes a marker so seeding never repeats — a default the user later
19    deletes stays gone.
20    """
21    marker = symlinks_dir / MARKER
22    if marker.exists():
23        return
15
The only public name. It takes the symlinks_dir to seed and returns None — it is called for its side effect, not a value.
22 — 23
The whole "run once" promise, in two lines. If .initialized exists, return immediately. No flags, no database, no config — the filesystem itself remembers.

Why a marker, and not "is the directory empty?"

If seeding triggered whenever symlinks/ looked empty, deleting the bundled docs from the index would simply bring them back on the next restart — an undeletable app. The marker decouples "have we ever seeded?" from "what is in here now," so a default the user deletes stays gone.

03 — the loop

Link what is missing, skip the rest.

Past the guard, it walks DEFAULT_APPS, creating a symlink for each — but only when it is safe to: never over an existing entry, never to a target that is not there.

05 — step back

Four things to take away.

The filesystem is the database

"Have we seeded?" is answered by a file existing, "what is registered?" by symlinks existing. There is no state store — consistent with the rest of the app, where symlinks/ is the only source of truth.

It respects deletion

Because the marker — not directory contents — gates seeding, an app you delete from the index never comes back. Seeding is a first-run convenience, not an enforced default.

It fails soft

An already-present slug is skipped; a missing target is a logged warning. The worst case is a log line, never a crash or a broken link on startup.

It is why this page exists on a fresh clone

The single entry in DEFAULT_APPS is this explainer. Clone, run, open localhost/ — and the tour is already registered, served by the very server it documents.
