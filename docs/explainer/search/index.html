01 — the filter

apply() — one query, three fields.

apply() lowercases the trimmed input, then walks every .row element — testing the query as a substring of the app name, card title, and description. An empty query matches everything. Non-matches vanish via style.display = 'none'.

search.jslines 7 — 16
7  function apply() {
8    const q = input.value.trim().toLowerCase();
9    document.querySelectorAll('.row').forEach((row) => {
10      const name  = (row.dataset.name || '').toLowerCase();
11      const title = (row.querySelector('.name')?.textContent || '').toLowerCase();
12      const desc  = (row.querySelector('.desc')?.textContent || '').toLowerCase();
13      const match = !q || name.includes(q) || title.includes(q) || desc.includes(q);
14      row.style.display = match ? '' : 'none';
15    });
16  }
8
trim().toLowerCase() before anything else — the query is normalised once, not per field. Trailing spaces and case differences never matter.
10 — 12
Three sources. data-name is the slug written directly into the HTML attribute — the filesystem name. .name is the display title. .desc is the one-line description. All three are lowercased to match the normalised query.
13
!q short-circuits for an empty input — every row is visible without touching the DOM. A non-empty query must appear as a substring of at least one field.
14
Resetting to '' (not 'block') lets the row's natural CSS display rule re-apply — the filter doesn't need to know how rows are laid out.

The quiet contract with keys.js and sort.js

keys.js's visibleRows() calls getComputedStyle(row).display !== 'none' to skip hidden rows during keyboard navigation. sort.js re-orders the DOM in place but leaves filtered rows hidden. The display toggle is the shared language — no events, no shared state.

02 — wiring

Two event listeners hook it up.

The input event fires on every keystroke. The clear button empties the field, re-applies the filter, and returns focus — so the keyboard shortcut stays live.

search.jslines 18 — 27
18  input.addEventListener('input', apply);
19
20  const clearBtn = document.querySelector('.search-clear');
21  if (clearBtn) {
22    clearBtn.addEventListener('click', () => {
23      input.value = '';
24      apply();
25      input.focus();
26    });
27  }
18
'input' — not 'keyup' or 'change'. input fires on every character, including paste and autocomplete; the filter is always current.
21
The if (clearBtn) guard mirrors the early-return at line 5 — the file is safe to load on any page that happens to share the stylesheet, even if the clear button is absent.
23 — 25
Clear: three steps in order — empty the value, re-apply so all rows reappear, then refocus. Refocusing means Esc or another / keypress is immediately handled without the user re-clicking.
03 — keyboard

Two global keys — / and Esc.

A single keydown listener on document handles both. The conditions are mutually exclusive — one focuses, the other blurs — so there is no ordering issue.

search.jslines 29 — 39
29  document.addEventListener('keydown', (e) => {
30    if (e.key === '/' && document.activeElement !== input) {
31      e.preventDefault();
32      input.focus();
33      input.select();
34    } else if (e.key === 'Escape' && document.activeElement === input) {
35      input.value = '';
36      apply();
37      input.blur();
38    }
39  });
30 — 31
The activeElement !== input guard prevents the shortcut firing when already inside the field. preventDefault() stops the slash from being typed into whatever had focus before — a text input on another widget, for instance.
33
select() highlights any existing text. A user who pressed / and starts typing replaces the previous query immediately — no backspacing required.
34 — 37
Esc only acts when the input is focused — it doesn't swallow the key globally. Clear, re-apply, then blur() — after which keyboard nav in keys.js (arrow keys, Enter) resumes naturally.

Forty lines, one responsibility

There is no state object, no event bus, no import. The file reads exactly as long as its job — wire an input to a DOM walk, add two keyboard shortcuts, ship it. The display toggle it sets is the only interface it publishes, and every sibling module reads it the same way.
