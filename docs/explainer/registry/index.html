One file, read closely

registry.py
turns a folder into data.

The symlinks/ directory is the only API the app has. This file walks that directory and hands back a list of App records — one per registered page. It is 89 lines of pure, side-effect-free reading: nothing here ever writes to disk, and nothing here ever raises.

02 — the record

The App frozen dataclass.

Every function in this file exists to produce one of these. It is the contract between registry.py and the Jinja template that renders the index — nine fields, and once built, none of them can change.

registry.pylines 17 — 27
17@dataclass(frozen=True)
18class App:
19    name: str
20    target: Path | None
21    title: str
22    description: str
23    icon: str            # raw value from meta.json (or default)
24    icon_svg: str        # ready-to-inline HTML
25    accent: str          # resolved hex
26    last_modified: datetime | None
27    error: str | None

The three | None fields are the tell. target, last_modified and error are exactly the fields that distinguish a healthy app from a broken one — a working app has a target and an mtime and no error; a broken one is the mirror image.

name
The symlink's filename. This is the app's identifier — it becomes the URL segment and is used to seed the accent hash.
target
The resolved directory the symlink points at, or None if the app is broken. A non-None target means the index.html exists.
title
Display name for the card. Defaults to name when meta.json supplies nothing.
description
One-line blurb under the title. Defaults to the empty string — the template simply renders nothing.
icon
The raw icon value as written in meta.json (or DEFAULT_ICON). Kept around unresolved — useful for debugging and round-tripping.
icon_svg
The resolved icon: inline-ready HTML produced by resolve_icon. Either a bundled Lucide SVG or an emoji span.
accent
A resolved #rrggbb string from resolve_accent. Never None — there is always a colour, even if only a hashed one.
last_modified
UTC timestamp of the app's index.html, or None when broken. Drives the "recently visited / recently changed" sort on the index.
error
A human-readable reason the app could not load, or None for a healthy app. Its presence is the definition of "broken".

Why frozen=True?

An App is a snapshot of the filesystem at the instant scan() ran. Freezing it makes that immutability enforced, not just intended — any code that holds an App and tries to mutate a field gets a FrozenInstanceError. It also makes the record hashable, so an App can live in a set or be used as a dict key for free. Cheap to gain, expensive to skip.

03 — the entry point

scan()  walks the directory.

The only public function. server.py calls it on every hit to GET /. It is deliberately thin — list the directory, delegate each entry to _load, return the list.

registry.pylines 30 — 38
30def scan(symlinks_dir: Path) -> list[App]:
31    if not symlinks_dir.exists():
32        return []
33    apps: list[App] = []
34    for entry in sorted(symlinks_dir.iterdir(), key=lambda p: p.name):
35        if entry.name.startswith("."):
36            continue
37        apps.append(_load(entry))
38    return apps
31 — 32
If the symlinks directory does not exist yet, return an empty list rather than raising. A first-run app with no symlinks/ folder still renders a valid (empty) index — degradation, not failure.
34
sorted(…, key=lambda p: p.name) makes the result deterministic. iterdir() yields entries in arbitrary filesystem order; sorting by name means the index card order is stable across requests and across machines.
35 — 36
Dotfiles are skipped — .DS_Store, .gitkeep, editor scratch. The directory holds app symlinks; anything beginning with . is treated as not-an-app and quietly ignored.
37
Every surviving entry becomes an App via _load — including broken ones. scan() never filters on health; a broken app is still a row in the list, so the index can show it.
04 — the error path

_broken()  the failure factory.

One helper, called from four places. It builds a valid App that nonetheless carries an error — the record shape never changes, only its contents.

registry.pylines 41 — 47
41def _broken(name: str, error: str) -> App:
42    return App(
43        name=name, target=None, title=name, description="",
44        icon=DEFAULT_ICON, icon_svg=resolve_icon(DEFAULT_ICON),
45        accent="#d6928f",  # broken rows always dusty rose
46        last_modified=None, error=error,
47    )
43
target=None and title=name — a broken app has nowhere to point and nothing to call itself, so it falls back to its raw filename. description is empty.
44
Even a broken row gets a real icon — resolve_icon(DEFAULT_ICON) yields the bundled box.svg. The card still has a glyph; it is simply the generic one.
45
The fixed accent. A healthy app's colour is hashed from its name and could be any of 32 pastels — but a broken app is hard-coded to #d6928f, a dusty rose. Note this is not any of the named accents in accents.py; it is a reserved off-palette colour so "broken" reads as visually distinct at a glance, never accidentally matching a healthy card.
46
last_modified=None (there is no index.html to stat) and error=error — the caller-supplied reason string is passed straight through and becomes the one piece of diagnostic the index can display.

Why a broken app is still an App

Returning _broken(…) instead of None means the list from scan() is uniform — every element is an App, and the template never has to special-case a missing entry. The difference between healthy and broken is data inside the record (error is not None), not the record's type.

05 — the workhorse

_load()  one entry to one App.

This is where a directory entry is validated and, if it survives, enriched. The structure is a ladder of guards: each one either returns a _broken app or falls through to the next.

registry.pylines 50 — 69
50def _load(link: Path) -> App:
51    name = link.name
52    try:
53        target = link.resolve(strict=True)
54    except (FileNotFoundError, OSError) as e:
55        return _broken(name, f"broken symlink: {e}")
56    if not target.is_dir():
57        return _broken(name, "symlink target is not a directory")
58    index = target / "index.html"
59    if not index.exists():
60        return _broken(name, "missing index.html")
61    meta = _load_meta(target / "meta.json", name)
62    last_modified = datetime.fromtimestamp(index.stat().st_mtime, tz=timezone.utc)
63    return App(
64        name=name, target=target,
65        title=meta["title"], description=meta["description"],
66        icon=meta["icon"], icon_svg=resolve_icon(meta["icon"]),
67        accent=resolve_accent(meta.get("accent"), name),
68        last_modified=last_modified, error=None,
69    )
52 — 55
Guard one. resolve(strict=True) follows the symlink and demands the target actually exist — if it does not, it raises. A dangling symlink becomes _broken(name, "broken symlink: …") with the exception's text appended for diagnosis. Catching OSError too covers permission errors and symlink loops.
56 — 57
Guard two. The target resolved, but it has to be a directory. A symlink pointing at a file is a broken app — an app is a folder of static assets, not a single file.
58 — 60
Guard three. The directory must contain an index.html. That file is the app's entry document; without it there is nothing for GET /{app}/ to serve, so the app is broken.
61
Past the guards. Metadata is delegated to _load_meta, which is given the path to a (possibly absent) meta.json and the name to use as the title default.
62
The freshness signal. The index.html's st_mtime is read and converted to a timezone-aware UTC datetime. Passing tz=timezone.utc explicitly avoids a naive local-time datetime — important once the value is compared or formatted.
63 — 69
The healthy App. title/description/icon come straight from the meta dict; icon_svg and accent are resolved here by the two sibling modules. Note meta.get("accent") may be None — and resolve_accent is built to accept exactly that, hashing the name into a colour instead.

The division of labour on lines 66 — 67

_load never decides what an icon or accent means. It passes the raw value (or None) to resolve_icon / resolve_accent and trusts them to apply policy — emoji fallback, Lucide lookup, hash-from-name. This file knows the filesystem; those files know the design system.

06 — the metadata reader

_load_meta()  never crashes on bad input.

An app may ship a meta.json to customise its card. It may also ship a broken one, an empty one, or none at all. This function's whole job is to absorb every one of those cases and still return a usable dict.

registry.pylines 72 — 89
72def _load_meta(path: Path, name: str) -> dict:
73    defaults = {"title": name, "description": "", "icon": DEFAULT_ICON, "accent": None}
74    if not path.exists():
75        return defaults
76    try:
77        data = json.loads(path.read_text())
78    except (json.JSONDecodeError, OSError) as e:
79        logger.warning("malformed meta.json at %s: %s", path, e)
80        return defaults
81    if not isinstance(data, dict):
82        logger.warning("meta.json at %s is not an object", path)
83        return defaults
84    out = dict(defaults)
85    for k in ("title", "description", "icon", "accent"):
86        v = data.get(k)
87        if isinstance(v, str) and v:
88            out[k] = v
89    return out
73
The defaults are built first, with name already baked in as the title. Every later branch can return this dict and the caller still gets all four keys it indexes on lines 65 — 67.
74 — 75
No meta.json at all is the common, expected case — not an error. Most apps never write one. Return the defaults silently, with no log line.
76 — 80
The never-crash core. A file that is invalid JSON, or unreadable, raises JSONDecodeError or OSError — both are caught, a warning is logged with the path and the exception, and the defaults are returned. One app's typo can never break the index.
81 — 83
Valid JSON is not enough — it must be a JSON object. A file containing [] or "hello" or 42 parses fine but is not a dict; it is logged and the defaults are used.
84
dict(defaults) makes a copy. The merge on the next lines mutates out in place — copying first keeps the shared defaults dict pristine for any later call.
85 — 88
A strict allow-list merge. Only the four known keys are even looked at — extra keys in meta.json are ignored. A value overrides its default only if it is a str and non-empty: isinstance(v, str) and v. A number, a null, or an empty string all leave the default in place.

Why only non-empty strings survive the merge

The four card fields are all rendered as text. Accepting "title": 7 or "icon": null would push a non-string into the App and downstream into the template. The and v half additionally rejects "" — an empty title is no title, so the name default is better. The result: every value in the returned dict is either the default or a real, non-empty string.

Read top to bottom, _load_meta is a ladder of four ways meta.json can fail to be useful — absent, unreadable, not an object, or carrying junk values — and each rung lands on the same safe floor:

1

File absent

No meta.json on disk → defaults, no log.

2

Unparseable or unreadable

Invalid JSON / I/O error → defaults, warning logged.

3

Not an object

Parsed to a list / scalar → defaults, warning logged.

4

Object with junk fields

Per-key: non-string or empty values silently fall back to the default.

07 — step back

Four things to take away.

This file is a pure reader

Nothing in registry.py writes, deletes, or mutates. It only observes the filesystem and returns immutable App records. That is what makes it safe to call on every single request to GET /.

Failure is data, not an exception

A broken symlink, a missing index.html, a corrupt meta.json — none of them raise. Each becomes either a _broken app or a fall-back to defaults. The list scan() returns is always complete and always uniform.

Broken always reads as dusty rose

Healthy cards get a name-hashed colour from a 32-pastel palette; broken cards are pinned to the off-palette #d6928f. The colour itself is the error indicator — no legend, no icon swap needed.

Policy lives in the siblings

_load decides what to resolve; accents.py and icons.py decide how. Keeping the filesystem walk and the design-system rules in separate files means each can change without disturbing the other.
