localhost-pages/technical-explanation served & auto-reloading via the app itself
A guided source tour

The source,
read line by line.

localhost-pages is an always-on local web server that gathers the HTML output of many small tools into one place — drop a symlink, get a card at localhost/, with live reload. Its entire source is twelve Python files and roughly 820 lines — plus the front-end it serves: a Jinja2 template, a stylesheet, eleven little scripts and a bundle of icons. This site gives each file its own page. And it is itself one of the registered apps — you are reading the docs through the machine they describe.

01 — before you read anything

Hold one sentence in your head.

“
The server is a thin shell over the filesystem. symlinks/ is the only source of truth — the registry just lists it, the server just serves it, the watcher just watches it, and SSE just tells browsers when to reload.
Every file below is one verb in that sentence. No hidden state, no cache, no database.
02 — the shape of it

How the pieces fit.

Three layers. On top, the world outside the process — the operating system, whose pf packet filter redirects :80 to the server's 127.0.0.1:8080 (so it never needs root), and the symlinks/ directory on disk. In the middle, the twelve-file Python server. Underneath, the front-end it renders and streams to your browser. Grey is the request–response path; the teal loop is live-reload.

request & response — a browser asks, the server renders live-reload loop — a file changes → watcher → pubsub → SSE → your tab reloads

05 — how to read it

A suggested path.

The whole project is one comfortable sitting. Pair each file with its tests/test_*.py — the tests double as executable documentation.

i

Orient — README.md & AGENTS.md

What it does, how to run it, and the symlink registration contract the whole server is built around.

ii

The spine — __main__.py, logconfig.py, server.py & bootstrap.py

The process entry point — which calls logconfig.py to turn on always-on rotating logs — then the file every route lives in, whose lifespan hook calls bootstrap.py to seed the bundled docs into symlinks/ on first run. Read create_app() top to bottom.

iii

The reload loop — pubsub.py, inject.py, watcher.py

Smallest first. Once these click, the SSE endpoint back in server.py reads trivially.

iv

The index pipeline — registry.py → index.py → icons.py → accents.py

Discover the apps, render the cards, give each one an icon and a deterministic colour.

v

The front-end — index.html & index.css, then the scripts

The template and stylesheet the server renders, then reload.js and the ten little scripts that make the index sortable, searchable and keyboard-driven — including lp-log.js, which forwards browser errors to the server. End on static/lucide/.

vi

The edges — tests/ & .superpowers/specs/

Each test file mirrors a module; the spec and plan docs explain the recent index redesign.
