01 — the whole file

Sixteen lines. Nothing hidden.

The entire script is an IIFE — immediately-invoked function expression — so it leaves no global variables. It reads its own data-app attribute, builds a URL, and calls connect() once to start the event stream.

reload.jslines 1 — 16
1(function () {
2  var script = document.currentScript;
3  var app = script && script.getAttribute("data-app");
4  if (!app) return;
5  var url = "/__events?app=" + encodeURIComponent(app);
6  var es;
7  function connect() {
8    es = new EventSource(url);
9    es.addEventListener("change", function () { location.reload(); });
10    es.onerror = function () {
11      es.close();
12      setTimeout(connect, 1000);
13    };
14  }
15  connect();
16})();
2 — 3
document.currentScript is valid while the script executes — it points to the <script> tag that inject.py spliced in. The data-app attribute on that tag is the app name, HTML-escaped by inject.py at serve time.
4
If data-app is absent the script exits immediately — a silent no-op. This makes the file safe to load in isolation, without a host page.
5
The SSE endpoint — /__events?app=<name>. The name is URI-encoded so any app name, even one with spaces or special characters, survives the query string intact.
8
new EventSource(url) opens a long-lived HTTP connection. The browser holds it open, reading newline-delimited frames as they arrive — the SSE protocol, native in every modern browser.
9
The only event the script cares about: change. When server.py's /__events route streams event: change\n, this listener fires and location.reload() refreshes the tab.
10 — 13
The error handler. es.close() tears down the dead connection; setTimeout(connect, 1000) schedules a fresh one a second later — an automatic reconnect loop with a one-second backoff.
02 — resilience

Four lines that keep the tab alive.

The onerror handler is small, but it handles every failure mode the developer actually encounters day-to-day.

An EventSource connection breaks in three common situations: the server restarts during a code change, the laptop sleeps and the OS kills idle connections, or an intermediate proxy closes the stream after a timeout. Without an error handler, the EventSource enters a browser-managed retry loop with exponential back-off — which is slow and not always reliable across all browsers.

With the explicit onerror on lines 10 — 13 the behaviour is predictable: close the stale object immediately, wait exactly one second, and call connect() again. The tab re-establishes its stream silently — no user action required, no stale state. The developer saves a file, the server restarts, and a second later the tab is listening again.

Why es.close() before setTimeout?

The browser can fire onerror while the underlying connection is still in a half-open state. Calling close() explicitly puts the EventSource into CLOSED state and prevents the browser's own reconnect logic from running in parallel with ours — so there is never more than one active connection at a time.

03 — the full loop

From file save to tab reload.

This script is the last step of a four-part chain. Each part hands off to the next through a well-defined interface — file events, a pubsub queue, an SSE frame, a browser API.

1

watcher.py detects

watchfiles fires a filesystem event. The watcher resolves which app owns the changed path and publishes the app's topic name to the shared PubSub broker.

2

pubsub.py fans out

PubSub.publish(topic) puts the message into every asyncio.Queue subscribed to that topic — one queue per open browser tab watching the same app.

3

server.py streams

The /__events route's stream() generator wakes, dequeues the message, and yields event: change\ndata: <topic>\n\n — a valid SSE frame — over the long-lived HTTP connection.

4

reload.js reloads

The EventSource delivers the frame as a JavaScript change event. The listener on line 9 fires: location.reload(). The tab refreshes. Loop complete.

Where inject.py fits in

inject.py's inject_reload_script() is called by server.py's file-serving route for every HTML response. It splices <script data-app="…" src="/__static/reload.js"></script> before </body>. The source file on disk is never touched — the injection happens at serve time, on the way out. This script is what that injected tag loads.
