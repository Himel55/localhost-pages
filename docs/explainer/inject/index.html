One file, read closely

inject.py
is the seam.

Sixteen lines — the smallest module in localhost-pages. It does exactly one thing: splice a single <script> tag into a page on its way out the door. The app's HTML on disk is never touched. This file is the entire reason "drop a symlink, get live reload" works.

01 — top of the file

One import, one constant.

No frameworks, no I/O. The module pulls in a single helper from the standard library and defines the marker string that the whole design pivots on.

inject.pylines 1 — 5
1from __future__ import annotations
2
3from html import escape
4
5MARKER = "data-localhost-pages-reload"
1
from __future__ import annotations defers evaluation of type hints to strings. It costs nothing and is applied uniformly across the package — the annotations on line 8 are read, never executed.
3
The only dependency: html.escape from the standard library. No third-party code, no template engine — escaping is the one piece of correctness this file cannot hand-roll safely.
5
MARKER is a constant, defined at module scope so it is computed once. It is both the attribute name stamped onto the injected tag and the sentinel the function searches for. A single string plays both roles — that is what makes the idempotency check on line 9 trivially correct.

Why an unusual attribute name?

data-localhost-pages-reload is a valid HTML5 data-* attribute, so the browser ignores it. It is also long and specific enough that no real page would ever contain that exact string by accident — which matters, because the function's only "have I already done this?" test is a plain substring search for it.

02 — signature & idempotency guard

Run twice, change nothing the second time.

The function takes a page and an app name, returns a page. Before doing any work it asks one question: is the marker already here?

inject.pylines 8 — 10
8def inject_reload_script(html: str, app_name: str) -> str:
9    if MARKER in html:
10        return html
8
A pure str → str transform. It takes html — the page text as read from disk — and app_name — the registered name of the app it belongs to. No defaults, no globals, no side effects: easy to test in isolation, easy to reason about.
9 — 10
The idempotency guard. If the marker substring already appears anywhere in the page, the function returns the input unchanged. MARKER in html is a plain substring search — cheap, and run before any allocation.

Why guard against double injection at all?

The function is the last step of serve_app in server.py, called on every HTML response. Nothing downstream stops a page that already carries the tag from passing through again — for instance an HTML file that was itself produced by a previous serve, or simply a defensive second call. Without the guard, a page could accumulate one reload script per request. Because the injected tag contains MARKER (see line 11), the very first injection makes every subsequent call a no-op. The function is idempotent: f(f(x)) == f(x).

03 — building the tag

One script element, escaped at the seam.

Past the guard, the function assembles the tag it will splice in. It is a literal <script> with three attributes — and exactly one of them is untrusted.

inject.pylines 11 — 12
11    tag = (f'<script {MARKER} src="/__static/reload.js" '
12           f'data-app="{escape(app_name, quote=True)}"></script>')
11
An f-string spread across two adjacent literals — Python concatenates them at compile time, so this is a single string with no runtime join. {MARKER} interpolates the constant from line 5 directly into the tag's attribute list. That is the move that makes the guard self-fulfilling: the tag the function emits is what the guard later finds.
11
src="/__static/reload.js" points at the package's own static mount. The __static prefix is the reserved namespace server.py sets up — user apps can never be named __…, so this path can never be shadowed.
12
data-app="…" carries the app name into the page so reload.js knows which SSE topic to subscribe to. This is the only attribute value built from data the function received as an argument — and it is the only one that is escaped.

Why escape(app_name, quote=True) is not optional

app_name originates from a path segment in an HTTP request. It is interpolated inside a double-quoted HTML attribute. html.escape turns &, < and > into entities — and crucially, quote=True additionally escapes the double quote character.

Without quote=True, an app name containing a " could close the data-app attribute early and inject arbitrary new attributes — or, with a >, break out of the tag entirely. That is a stored-XSS hole opened by nothing more than the name on a symlink. The default for html.escape is quote=True, but passing it explicitly states the intent: this value lands between quotes, so quotes must die.

04 — finding the splice point

Locate </body>, or fall back.

The tag has to go somewhere. The function looks for the closing body tag — searching from the end — and has a defined answer for when there isn't one.

inject.pylines 13 — 15
13    idx = html.rfind("</body>")
14    if idx == -1:
15        return html + tag
13
rfind — not find — returns the index of the last occurrence of </body>. A well-formed document has exactly one, so for valid input the direction is irrelevant. But searching from the end is the safer choice: if a page contains the literal text </body> earlier (inside a code sample, a comment, an inline script), rfind still targets the structural closing tag, which is the last one.
14 — 15
The fallback. rfind returns -1 when the substring is absent. Rather than fail, the function appends the tag to the very end of the document and returns. A fragment with no </body> — a partial template, a bare snippet — still gets a working reload script. Degraded placement, never a crash.

Why splice before </body>?

Putting the script at the bottom of the body is the conventional spot for behavioural JavaScript: by the time the browser reaches it, the document above has been parsed. It also keeps the injection visually unobtrusive and out of the page author's <head>. The function does a textual splice, not a DOM parse — it never builds a tree, so this string-position approach is the cheapest correct way to land the tag in the right place.

05 — the splice

Slice, insert, return.

With an index in hand, the final line rebuilds the page in three pieces and hands it back.

inject.pyline 16
16    return html[:idx] + tag + html[idx:]
16
Three slices concatenated: everything before the closing tag, then the new tag, then </body> and everything after it. Because html[idx:] starts at the index, the closing tag is preserved — the script is inserted immediately in front of it, exactly as planned in section 04.
16
The original html string is never mutated — Python strings are immutable. A brand-new string is built and returned. The input the caller passed in is left exactly as it was.

The on-disk file is never written

Every path out of this function — the guard on line 10, the fallback on line 15, the splice on line 16 — returns a string. The module has no open, no write, no Path. It cannot modify a file even if it wanted to. The rewrite exists only in the response body that server.py sends back over the wire.

06 — where it sits

The seam between disk and wire.

This function has exactly one caller. It is the last thing serve_app does to an HTML response before returning it — line 117 of server.py:

server.pylines 115 — 117  (for context)
115        if ctype == "text/html":
116            html = requested.read_text(encoding="utf-8", errors="replace")
117            return HTMLResponse(inject_reload_script(html, app_name))

The file is read from disk into a string, that string is passed straight into inject_reload_script, and the result becomes the response body. The bytes on disk and the bytes on the wire diverge at exactly this point — and the only difference between them is one <script> tag.

On disk — untouched

<body> <h1>my app</h1> </body>
→

On the wire — spliced

<body> <h1>my app</h1> <script data-localhost-pages-reload …> </body>

Sixteen lines that make the whole app feel alive

The reload script this function inserts opens an EventSource to /__events, listens for the change events that watcher.py publishes through pubsub.py, and calls location.reload(). None of that lives here — but none of it would reach the page without this splice. inject.py is the seam that connects a plain HTML file an author wrote to the live-reload machinery they never asked for.

07 — step back

Four things to take away.

The file is the live-reload magic — and it is pure

No I/O, no state, no globals. A str → str function and one constant. The smallest module in the package carries the feature the whole project is named for, precisely because it does so little: it only rewrites a string.

One constant, two jobs, idempotent for free

MARKER is stamped into the tag and searched for as a sentinel. Because those are the same string, the first injection guarantees every later call short-circuits — f(f(x)) == f(x) falls out of the design with no extra bookkeeping.

Escaping is the one thing it must not get wrong

app_name comes from an HTTP request and lands inside a quoted attribute. escape(app_name, quote=True) is what stops a maliciously named symlink from breaking out of the tag. The one untrusted value is the one value that is escaped.

Every code path returns a valid page

Marker present → return as-is. No </body> → append. Normal page → splice. There is no exception, no None, no error case. Whatever HTML goes in, working HTML comes out.
