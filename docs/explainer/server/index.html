02 — the factory

create_app() builds, never owns.

The whole file is one function. It returns a configured FastAPI app — it does not create a module-level app. That single choice is what makes the app testable.

server.pylines 47 — 65
47def create_app(symlinks_dir: Path = DEFAULT_SYMLINKS) -> FastAPI:
48    pubsub = PubSub()
49    watcher = Watcher(symlinks_dir, pubsub)
50
51    @asynccontextmanager
52    async def lifespan(_app: FastAPI):
53        symlinks_dir.mkdir(parents=True, exist_ok=True)
54        seed_defaults(symlinks_dir)
55        apps = scan(symlinks_dir)
56        logger.info("apps registered: %d (%d broken)",
57                    len(apps), sum(1 for a in apps if a.error))
58        await watcher.start()
59        try:
60            yield
61        finally:
62            await watcher.stop()
63
64    app = FastAPI(lifespan=lifespan)
65    app.mount("/__static", StaticFiles(directory=STATIC_DIR), name="static")
48 — 49
Per-app state, created fresh each call: a PubSub broker and a Watcher, wired together — the watcher will publish onto this exact pubsub.
52 — 62
lifespan is FastAPI's startup/shutdown hook. It ensures symlinks/ exists, seeds the default apps, then scans and logs the app count (apps registered: N (M broken)), starts the watcher, and — via try/finally — guarantees the watcher is stopped cleanly on shutdown.
65
The package's static assets mount under /__static. The __ prefix is a reserved namespace — note that the serve route below explicitly refuses any app named __…, so user content can never shadow it.

Why a factory and not app = FastAPI()?

Because symlinks_dir is a parameter. Each test can call create_app(symlinks_dir=tmp_path) and get an app pointed at a throwaway directory — no global state, no interference between tests. See tests/conftest.py.

Plus one thing that isn't a route  — the request middleware (lines 67 — 84)

An @app.middleware("http") wraps every request: it times the call and logs METHOD path -> status (Nms) (static-asset hits drop to DEBUG), and on any unhandled exception it logs a full traceback before re-raising. It's part of the HTTP surface even though it owns no path — see the observability section below.

04 — observability

Everything that happens gets written down.

Three pieces conspire so that when something breaks, there is a record: a middleware that logs every request and every unhandled exception, a route that catches browser errors into the same log, and a route that hands the user a downloadable bundle of it all.

server.pylines 67 — 84  (the middleware)
67    @app.middleware("http")
68    async def log_requests(request: Request, call_next):
69        start = time.perf_counter()
70        try:
71            response = await call_next(request)
72        except Exception:
73            dur = (time.perf_counter() - start) * 1000
76            logger.exception("unhandled error %s %s after %.0fms",
77                             request.method, request.url.path, dur)
78            raise
79        dur = (time.perf_counter() - start) * 1000
81        level = logging.DEBUG if request.url.path.startswith("/__static") else logging.INFO
82        logger.log(level, "%s %s -> %d (%.0fms)",
83                   request.method, request.url.path, response.status_code, dur)
84        return response
69 — 84
Every request is timed with perf_counter and logged as METHOD path -> status (Nms). The 1700+ static-asset hits are noisy, so those drop to DEBUG while real requests stay at INFO.
72 — 78
The catch-all that earns its keep: any unhandled exception is logged with logger.exception — a full traceback — before being re-raised. These would otherwise scroll off the terminal and vanish.

One click, one attachable file

The bundle is plain text: version + timestamp + Python/platform, then the app inventory (each app marked ok or BROKEN), then the tail of the log file — bounded to _MAX_LOG_CHARS so it stays small enough to email. The Content-Disposition: attachment header makes the browser download it as localhost-pages-diagnostics.txt — exactly what to staple to a bug report.

05 — route one

GET /  the index.

server.pylines 129 — 131
129    @app.get("/", response_class=HTMLResponse)
130    async def index() -> HTMLResponse:
131        return HTMLResponse(render_index(scan(symlinks_dir)))

Two calls, read inside-out. scan(symlinks_dir) walks the symlinks directory and returns a list of App records. render_index(…) feeds that list to a Jinja2 template and returns HTML.

There is no cache — every hit re-scans the disk. At localhost scale that is free, and it means the index always reflects the filesystem exactly as it is right now.

06 — route two

GET /__events  the reload channel.

This is how an open tab learns a file changed. The response never ends — it's a Server-Sent-Events stream the browser holds open.

server.pylines 133 — 150
133    @app.get("/__events")
134    async def events(app: str = Query(default="")):
135        topic = app or INDEX_TOPIC
136
137        async def stream():
138            q = pubsub.subscribe(topic)
139            try:
140                yield ": connected\n\n"
141                while True:
142                    try:
143                        msg = await asyncio.wait_for(q.get(), timeout=15.0)
144                        yield f"event: change\ndata: {msg}\n\n"
145                    except asyncio.TimeoutError:
146                        yield ": ping\n\n"
147            finally:
148                pubsub.unsubscribe(topic, q)
149
150        return StreamingResponse(stream(), media_type="text/event-stream")
134 — 135
?app=foo subscribes to that app's topic; absent, you get INDEX_TOPIC — the channel the index page listens on.
138
subscribe() hands back an asyncio.Queue. The watcher will drop the topic name into it whenever a matching file changes.
140
A line starting with : is an SSE comment — the browser ignores it, but sending it flushes the headers so the connection is confirmed open.
143 — 146
The heart. Either an event arrives within 15 s and is forwarded as a proper event: change frame, or the wait times out and a keep-alive ping is sent — which stops idle proxies and browsers from dropping the connection.
147 — 148
finally runs when the client disconnects — the queue is removed from the pubsub so subscriptions never leak.

The wire format

event: change\ndata: <topic>\n\n is the SSE protocol. The browser's EventSource API delivers it as a JavaScript change event — which the injected reload.js listens for, then calls location.reload().

07 — routes three & four

DELETE /__apps/…  unregister.

The "✕" button on a card. Almost the entire body is validation — deleting by a name from an HTTP request is exactly where path-traversal bugs live.

152    @app.delete("/__apps/{name}", status_code=204)
153    async def delete_app(name: str):

165            logger.info("deleted app %r", name)
166            return
167        raise HTTPException(status_code=404, detail="not found")
169    @app.delete("/__apps/{path:path}", status_code=400)

171        raise HTTPException(status_code=400, detail="invalid app name")
154
First filter — a cheap string check. Empty names, the reserved __ prefix, slashes, and .. are all rejected before the filesystem is touched.
157 — 158
The load-bearing check. Even after the string filter, _is_within(path.parent, symlinks_dir) resolves the parent and asserts it is still inside symlinks_dir. This catches a symlinked parent that could otherwise redirect the deletion elsewhere.
159 — 166
Only unlink() if it is genuinely a symlink — a real directory is never deleted. The unlink is wrapped: an OSError is logged (logger.error) then surfaced as a 500, and a clean delete is logged (logger.info) before returning.
169 — 171
A routing safety net. The first route matches a single path segment; anything containing a slash falls through to this {path:path} catch-all and is rejected with 400.
08 — route five

GET /{app_name}/{path:path}

The catch-all that serves every registered app's files. It locates a file safely, then decides whether to rewrite it. This is the longest route — and the most security-critical.

server.pylines 173 — 203  (condensed)
173    @app.get("/{app_name}/{path:path}")
174    async def serve_app(app_name: str, path: str):
175        if app_name.startswith("__"):  # guard 1
176            raise HTTPException(status_code=404)
177        link = symlinks_dir / app_name
181        target = link.resolve(strict=True)   # guard 2
184        if not target.is_dir(): raise HTTPException(404)
187        rel = path or "index.html"           # dir-index default
192        requested = (target / rel).resolve(strict=True)  # guard 3
195        if not _is_within(requested, target): raise HTTPException(404)  # guard 4
200        if requested.suffix.lower() in HTML_SUFFIXES:
201            html = requested.read_text(encoding="utf-8", errors="replace")
202            return HTMLResponse(inject_reload_script(html, app_name))
203        return FileResponse(requested)

The first two-thirds of this route is four independent safety checks. Any one alone would leave a directory-traversal hole; stacked, they hold:

1

Reserved prefix

Line 175 — a name starting with __ is refused, so user content can never shadow /__events or /__static.

2

Strict symlink resolve

Line 181 — resolve(strict=True) raises if the link or its target doesn't exist. A broken symlink becomes a clean 404, never a crash.

3

Strict file resolve

Line 192 — the full requested path is resolved with strict=True; any missing component (including .. games) raises and 404s.

4

Containment

Line 195 — after all symlinks resolve, _is_within(requested, target) must hold. If the real path escaped the app's directory, it 404s. The decisive guard.

Lines 200 — 203  — the one piece of magic

Content negotiation. If the file's suffix is in HTML_SUFFIXES (.html/.htm), it is read as text and passed through inject_reload_script(), which splices the SSE client in before </body>. Anything else streams untouched via FileResponse. This is why "drop a symlink and get live reload" works — the app's HTML on disk is never modified; the reload script is added at serve time, on the way out.

09 — step back

Four things to take away.

The __ prefix is the entire framework/user boundary

Internal routes live under __events, __apps, __static; serve_app refuses any app named __…. That naming convention is the isolation — there is nothing else enforcing it.

Security is layered, never single-checked

String filters, parent resolution, strict=True resolves, and relative_to containment. Defense in depth — the file server assumes every earlier check could be bypassed.

No caching, anywhere

Every request re-reads the filesystem — scan() on the index, a fresh read_text() per file. At localhost scale that is free, and the server always mirrors disk state exactly.

The factory is what makes it testable

create_app(symlinks_dir=…) takes the one piece of external state as a parameter. No module-level globals — so the test suite spins up isolated apps freely.
