peek.js
a preview without leaving.

Click any app row and a centered iframe modal slides in — no navigation, no new tab. The modal is history-aware: one history.pushState is all it takes, so the browser Back button closes it cleanly. Modifier-clicks and the explicit ↗ button always escape to the real page.

01 — building the modal

build() is lazy — first call only.

The DOM nodes are created exactly once, on the first click that needs them. After that, if (modal) return makes every subsequent call free.

peek.jslines 10 — 43
10  function build() {
11    if (modal) return;
12    backdrop = document.createElement('div');
13    backdrop.className = 'peek-backdrop';
14    modal = document.createElement('div');
15    modal.className = 'peek-modal';
16    modal.innerHTML =
17      '<a class="peek-open" href="#" aria-label="Open full page">' +
24      '<button class="peek-close" type="button">×</button>' +
25      '<iframe sandbox="allow-same-origin allow-scripts"' +
         '        referrerpolicy="no-referrer"></iframe>';
26    backdrop.appendChild(modal);
27    document.body.appendChild(backdrop);
36    modal.querySelector('.peek-open').addEventListener('click', (e) => {
37      const a = e.currentTarget;
38      if (e.metaKey || e.ctrlKey || e.shiftKey || e.altKey) return;
39      if (e.button !== undefined && e.button !== 0) return;
40      e.preventDefault();
41      window.location.replace(a.href);
42    });
43  }
11
The guard. modal is module-level; once assigned it is truthy and build() becomes a no-op. One DOM tree, for the entire session.
12 — 15
Two nodes: backdrop (full-screen dimmed overlay) and modal (the centered card). The backdrop click-handler closes the modal when you click outside.
25
The iframe carries sandbox="allow-same-origin allow-scripts" — it can run the app's own scripts and read its own cookies, but form submission, popups, and top-level navigation are blocked by default. referrerpolicy="no-referrer" keeps the app unaware it is being previewed.
36 — 42
The ↗ maximize handler uses window.location.replace, not href navigation. replace overwrites the current history entry — so pressing Back from the full app returns to the bare index, not a bfcache-restored open preview.
02 — opening

open(row) — one pushState, one replace.

Two deliberate choices about how the iframe loads keep the browser history clean and the visit counters honest.

peek.jslines 45 — 67
45  function open(row) {
46    if (!row || !row.dataset.name) return;
47    if (row.classList.contains('broken')) return;
48    if (document.body.classList.contains('editing')) return;
49    build();
50    openRow = row;
51    const url = '/' + encodeURIComponent(row.dataset.name) + '/';
55    const iframe = modal.querySelector('iframe');
56    try {
57      iframe.contentWindow.location.replace(url);
58    } catch {
59      iframe.src = url;
60    }
61    modal.querySelector('.peek-open').href = url;
62    backdrop.classList.add('open');
63    history.pushState({ peek: row.dataset.name }, '');
66    window.__lpRecordVisit?.(row);
67  }
46 — 48
Three early exits: no data-name, broken symlink, or body in editing mode (rename in progress). None of these should preview.
57
The key choice. iframe.contentWindow.location.replace(url) rather than setting iframe.src. Using replace keeps the iframe navigation out of the top-level back stack entirely — so a single history.pushState on line 63 is all the Back button has to pop. Setting iframe.src would add its own entry and make Back unreliable.
58 — 59
The try/catch fallback. Cross-origin iframes block contentWindow.location access — in that case, assigning iframe.src is the only option.
63
pushState({ peek: name }, '') adds exactly one history entry. The state carries the app name so a future popstate can identify it.
66
window.__lpRecordVisit?.(row) — optional chaining on a global hook defined by visited.js. Opening a preview counts as a visit: it updates the recently-visited sort and clears the has-update badge.
03 — closing & history

close() balances the history stack.

Every open adds one entry; every close pops it — unless the close was triggered by the Back button itself, in which case the entry is already gone.

peek.jslines 69 — 85
69  function close(fromPopstate = false) {
70    if (!backdrop) return;
71    if (!backdrop.classList.contains('open')) return;
72    backdrop.classList.remove('open');
73    const iframe = modal.querySelector('iframe');
74    try {
75      iframe.contentWindow.location.replace('about:blank');
76    } catch {
77      iframe.src = 'about:blank';
78    }
79    openRow = null;
80    if (!fromPopstate && history.state?.peek) history.back();
81  }
83  window.addEventListener('popstate', () => {
84    if (backdrop?.classList.contains('open')) close(true);
85  });
70 — 71
Two guards. If build() was never called, backdrop is null. If it was called but nothing is open, removing open again would be harmless — but the history.back() on line 80 would incorrectly pop an unrelated entry.
75
Resetting to about:blank stops the iframe's network activity and frees the page from memory. Same replace-vs-src logic as in open().
80
The history balance. When the × button triggers close(), fromPopstate is false — so history.back() is called to pop the entry that open() pushed. When the Back button triggers popstate, the entry is already gone — so fromPopstate = true skips the extra back() that would otherwise eat an unrelated entry.
83 — 85
The popstate listener is the bridge between the browser's Back button and close(). It fires whenever the history stack is navigated — if the modal is currently open, close it.

Why check history.state?.peek on line 80?

If the user opens then immediately closes before the pushState completes — or if the modal was somehow shown without going through open() — the peek state may not be present. Checking avoids popping a history entry that peek did not push.

04 — click interception

A delegated handler — and six ways to stand down.

A single click listener on document catches every row click. Before calling open() it checks six conditions; any one lets the native click through.

peek.jslines 87 — 101
87  document.addEventListener('click', (e) => {
90    if (e.metaKey || e.ctrlKey || e.shiftKey || e.altKey) return;
91    if (e.button !== undefined && e.button !== 0) return;
93    if (e.target.closest('.open, .pin, .del')) return;
94    const row = e.target.closest('.row');
95    if (!row) return;
96    if (row.classList.contains('broken')) return;
97    if (document.body.classList.contains('editing')) return;
98    if (row.hasAttribute('data-confirming')) return;
99    e.preventDefault();
100    open(row);
101  });
1

Modifier keys

Line 90 — cmd/ctrl/shift/alt means the user wants "open in new tab" or similar. Let the underlying <a class="body"> link handle it natively.

2

Middle-click

Line 91 — e.button !== 0 catches middle-button and right-button clicks. Middle-click opens a new tab; peek should not intercept that.

3

Action buttons

Line 93 — clicks on .open, .pin, or .del belong to their own handlers. closest() means clicking an icon inside the button is still caught.

4

Broken row

Line 96 — a broken symlink has no app to preview. The row renders with a broken class; peek stands down and lets the row's default style communicate the error.

5

Edit mode

Line 97 — when the body carries editing, a rename input is active. Clicking the row should not accidentally open the preview.

6

Delete confirm

Line 98 — data-confirming means delete.js is waiting for the user to confirm deletion. That interaction takes priority; peek leaves it alone.

05 — keyboard

Two keys: Esc closes, f maximizes.

The keydown listener is a no-op unless the backdrop is open — so it costs nothing when no preview is showing.

peek.jslines 103 — 117
103  document.addEventListener('keydown', (e) => {
104    if (!backdrop?.classList.contains('open')) return;
105    if (e.key === 'Escape') {
106      e.stopImmediatePropagation();
107      close();
108      return;
109    }
110    if (e.key === 'f' || e.key === 'F') {
111      const tag = e.target.tagName;
112      if (tag === 'INPUT' || tag === 'TEXTAREA' || tag === 'SELECT') return;
113      e.preventDefault();
114      const link = modal.querySelector('.peek-open');
115      if (link?.href) window.location.replace(link.href);
116    }
117  });
104
Optional-chaining on backdrop — if build() has never run, backdrop is null and the whole listener exits in one expression.
106
stopImmediatePropagation — not just stopPropagation. Other listeners on document (search, keyboard shortcuts in the main UI) may also claim Esc. Peek wins when it is open; no other handler sees the event.
111 — 112
The input guard for f. If focus is inside a form element, the user is typing — f is just the letter f. The guard prevents maximizing while the user is filling in a field inside the previewed app.
115
Same window.location.replace path as the ↗ button handler. The history entry is overwritten, not pushed — Back leads to the bare index.
