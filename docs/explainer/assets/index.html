01 — what's bundled & why

Vendored in full, on purpose.

An offline-first localhost tool has no business phoning home for an icon.

Lucide is an open-source icon library licensed under the ISC licence — permissive, attribution-free in practice. The entire set ships inside src/localhost_pages/static/lucide/ as 1703 plain .svg files, one per icon, named by slug (rocket.svg, terminal.svg, …).

Vendoring the whole library means an app's meta.json can name any icon and it resolves — no network round-trip, no missing-icon surprise in a hot-spot, no version drift when Lucide publishes a new release upstream.

The font, JetBrains Mono, follows the same principle: one woff2 file on disk, referenced by the package's own index.css. The browser never needs to reach out.

Why not a CDN?

localhost-pages runs on developer machines, often offline or behind strict firewalls. Vendoring is the only reliable strategy — if the file is in the repo, it is always there.

No build step

The SVGs are served raw by FastAPI's StaticFiles mount at /__static. No bundler, no sprite sheet, no compile phase — just files and an HTTP server.

03 — icons.py

resolve_icon() — four branches.

icons.py is the only code that touches the bundle. It exposes one public function and keeps a private in-memory cache.

^[a-z0-9][a-z0-9-]*$ — must start with a letter or digit, then only lowercase, digits, or hyphens. No slashes, no dots, no path tricks.
13
_cache is a module-level dict. Once a file is read it is never read again for the lifetime of the process — icon resolution is effectively free after the first hit.
36 — 37
Branch 1. None, empty string, or any falsy value maps to box.svg — the visual default for apps without a declared icon.
39 — 40
Branch 2. A value starting with <svg is returned verbatim — caller-supplied markup, trusted by the operator who wrote meta.json.
42 — 50
Branch 3. A valid Lucide slug loads its file. If the named icon is absent — possible if a custom Lucide build is used — the fallback chain is: named icon → box.svg → empty <span>.
51
Branch 4. Anything else — an emoji, a URL, arbitrary text — is HTML-escaped and wrapped in a .icon-emoji span. Safe by default.

The regex is also a security boundary

^[a-z0-9][a-z0-9-]*$ rejects anything that could be used to escape LUCIDE_DIR: no /, no .., no uppercase, no extension. The name is safe to concatenate with a path without further checks.

04 — icon file shape

What's actually inside a lucide/*.svg.

Every file in the bundle shares the same outer shell. Only the inner paths differ.

fill="none" — the icon is drawn entirely in stroke, never filled. stroke="currentColor" means the icon inherits whatever CSS color its parent sets — no inline colour, no JavaScript, no filter hack needed to recolour it.

The 24×24 grid and the 2 px stroke are the Lucide design contract. Every icon in the 1703-strong set obeys them, which is why they can be mixed freely and still look like a single family.

stroke
currentColor means: whatever color CSS property is in effect at the call site. Set color: red on a parent <div> and the icon goes red — no additional CSS required.
fill
none — these are outline icons, not filled shapes. The visual weight comes entirely from the stroke, which makes them look consistent at any CSS font-size scale.
viewBox
0 0 24 24 — scale freely with width and height CSS properties; the viewport stays proportional.
05 — the font

One file, every weight.

JetBrains Mono ships as a single woff2 — a variable font covering the full 100 900 weight axis.

Format: woff2 (variable) Axes: wght 100–900 File size: ~90 KB font-display: swap

The font is declared in the package's own index.css via a single @font-face rule pointing at /__static/JetBrainsMono.woff2. font-weight: 100 900 tells the browser the file covers the full range; font-display: swap ensures text is immediately visible in a fallback font while the file loads — though at localhost latency, the swap is imperceptible.

A variable font replaces what used to require six or more separate files (one per weight). One file, one network request, every weight available in CSS with font-weight.

Why JetBrains Mono?

It is a programmer's monospace with generous spacing, clear disambiguation of 0/O and l/I/1, and an open licence (OFL). The UI is built for developers reading paths, names, and code — the font fits the context.
