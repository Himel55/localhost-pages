01 — the five modes

One dropdown. Five orderings.

The file's own header comment names them. Each mode reads different data — some from data-* attributes on the DOM, others from localStorage.

alpha
Sorts by data-name ascending — plain alphabetical order via localeCompare. The fallback when nothing has been chosen.
recently-updated
Sorts by data-mtime descending — file modification time stamped server-side and rendered into each row's HTML attribute.
recently-visited
Sorts by lp:visited:<name> ISO timestamp descending — written to localStorage by visited.js each time a card is clicked.
most-visited
Sorts by lp:visits:<name> integer counter descending — the running click count from visited.js. Name is the tiebreaker.
manual
Sorts by position in the lp:order JSON array written by edit.js. Apps not in the array sort last via Number.MAX_SAFE_INTEGER.

Legacy migration — 'recent'

Lines 25–26 handle a rename. The stored value 'recent' is silently promoted to 'recently-updated' so existing users keep their chosen sort after deploy — no jarring reset to alpha.

02 — the comparators

compare(m) returns the right comparator.

One factory function; one branch per mode. The return value is passed directly to Array.prototype.sort.

sort.jslines 43 — 65
43  function compare(m) {
44    if (m === 'recently-updated') {
45      return (a, b) => (b.dataset.mtime || '').localeCompare(a.dataset.mtime || '');
46    }
47    if (m === 'recently-visited') {
48      return (a, b) => visitedAt(b.dataset.name).localeCompare(visitedAt(a.dataset.name))
49        || a.dataset.name.localeCompare(b.dataset.name);
50    }
51    if (m === 'most-visited') {
52      return (a, b) => visitCount(b.dataset.name) - visitCount(a.dataset.name)
53        || a.dataset.name.localeCompare(b.dataset.name);
54    }
55    if (m === 'manual') {
56      const order = manualOrder();
57      const idx = (n) => {
58        const i = order.indexOf(n);
59        return i === -1 ? Number.MAX_SAFE_INTEGER : i;
60      };
61      return (a, b) => idx(a.dataset.name) - idx(b.dataset.name)
62        || a.dataset.name.localeCompare(b.dataset.name);
63    }
64    return (a, b) => a.dataset.name.localeCompare(b.dataset.name);
65  }
45
localeCompare on ISO timestamps works because the format is lexicographically ordered. Reversed (b before a) gives newest-first. Missing values coerce to '', which sorts last.
48 — 49
The || at line 49 is the tiebreaker: when two apps have identical timestamps (both unvisited), name alphabetical order decides. The same pattern repeats for most-visited at line 53.
56 — 59
Manual mode builds an index map once, before returning the comparator. manualOrder() parses lp:order from JSON; names absent from the array receive Number.MAX_SAFE_INTEGER and float to the bottom.
64
The alpha fallthrough — the final return outside all branches. Any unrecognised mode value resolves here rather than crashing.
03 — apply & renumber

Pinned rows always lead.

apply() splits rows into two groups, sorts each independently, then appends them back in order: pinned first, then the rest. renumber() patches the visible two-digit labels to match.

sort.jslines 67 — 82
67  function renumber() {
68    let i = 1;
69    list.querySelectorAll('.row').forEach((row) => {
70      const num = row.querySelector('.num');
71      if (num) num.textContent = String(i++).padStart(2, '0');
72    });
73  }
74
75  function apply() {
76    const cmp = compare(mode);
77    const rows = Array.from(list.querySelectorAll('.row'));
78    const pinned = rows.filter((r) => r.classList.contains('pinned')).sort(cmp);
79    const rest = rows.filter((r) => !r.classList.contains('pinned')).sort(cmp);
80    [...pinned, ...rest].forEach((r) => list.appendChild(r));
81    renumber();
82  }
78 — 79
Two passes through the same rows array — one for the pinned group, one for the rest. Each is sorted with the same comparator, so the chosen mode applies within each tier independently.
80
Spread, concatenate, appendChild in order. Because appendChild moves existing nodes, this is a zero-flicker DOM reorder — no elements are created or destroyed.
68 — 72
renumber() walks the list after the DOM has been reordered and rewrites every .num label with padStart(2, '0') — so the visible counter always reflects the current order, not the server-rendered one.
04 — the dropdown

Hand-built. Four interaction paths.

No native <select>. Visibility is controlled via menu.hidden and the aria-expanded attribute — CSS rotates the caret on the latter. Four distinct gestures close or open it.

93  function openMenu() {
94    menu.hidden = false;
95    toggle.setAttribute('aria-expanded', 'true');
96  }
97  function closeMenu() {
98    menu.hidden = true;
99    toggle.setAttribute('aria-expanded', 'false');
100  }

102  toggle.addEventListener('click', (e) => {
103    e.stopPropagation();

107  menu.addEventListener('click', (e) => {

112    closeMenu();
113  });
114
115  document.addEventListener('click', (e) => {
116    if (!menu.hidden && !wrap.contains(e.target)) closeMenu();

119  document.addEventListener('keydown', (e) => {
120    if (menu.hidden) return;
121    if (e.key === 'Escape') {

102 — 105
Toggle on button click. stopPropagation prevents the document-level click handler (line 115) from immediately closing the menu that was just opened — the classic click-outside pattern requires this guard.
107 — 113
Select on item click. closest('li[data-value]') means any click inside a list item — even on a child element — hits the right target. stopPropagation again, for the same reason.
115 — 117
Click-outside-to-close. wrap.contains(e.target) is the test — if the click landed anywhere outside the dropdown component, close it.
119 — 126
Esc-to-close. stopImmediatePropagation — stronger than stopPropagation — prevents keys.js from also acting on the Escape key while the menu is open. Focus is returned to the toggle button afterward.

aria-expanded does double duty

It is both an accessibility attribute — screen readers announce the open/closed state — and a CSS hook. The stylesheet's caret rotation is driven by [aria-expanded="true"] .caret { transform: rotate(180deg); }. No extra class toggle needed.

05 — hooks & persistence

One public hook. Always fresh.

setMode persists the chosen sort to localStorage on every change. The module exposes itself to sibling scripts via a single property on the DOM node — and re-runs apply on every page restore.

sort.jslines 84 — 91, 128 — 133
84  function setMode(value, persist = true) {
85    mode = value;
86    if (persist) localStorage.setItem('lp:sort', value);
87    const item = items.find((i) => i.dataset.value === value);
88    current.textContent = item ? item.textContent : value;
89    items.forEach((i) => i.setAttribute('aria-selected', String(i.dataset.value === value)));
90    apply();
91  }
128
129  wrap.__setSort = setMode;

132  window.addEventListener('pageshow', apply);
84
The persist flag defaults to true for user-initiated changes but is passed as false at initialisation (line 28) — so the stored value is restored without being immediately re-written over itself.
88 — 89
setMode keeps the UI consistent: it updates the button label and sets aria-selected on every list item so the currently active option is visually and semantically marked.
129
The public hook. Attaching __setSort to the DOM node — rather than exposing a global — keeps the API scoped to the component. edit.js and pin.js call it to switch to 'manual' after a drag-drop or pin action respectively.
132
pageshow fires on both normal loads and bfcache restores. Re-running apply here means a recently-visited sort stays correct even when the user navigates away and returns with the back button — localStorage may have been updated since the page was first rendered.

Why attach to the DOM node and not window?

All five interaction scripts are IIFEs — they declare nothing global. Attaching __setSort to wrap (the .sort-wrap element) gives sibling scripts a predictable handle without polluting the global scope. The leading __ signals "internal contract, not public API".

Freshness sorts and SSE reloads

When the watcher fires a change event, the injected reload.js calls location.reload(). That triggers pageshow, which calls apply — so a recently-updated sort re-evaluates the new data-mtime values without any extra wiring.
